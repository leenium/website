user@leenium:~$ install

No pre-built ISO yet — you build it yourself or run the installer on existing Arch. Three ways to do it.

Method 1 — Build the ISO yourself

Wipes drive

Needs Docker. Builds a bootable image that handles partitioning, base Arch, and Leenium in one shot. Flash it, boot it, done.

# Requires Docker
git clone https://github.com/leenium/iso
cd iso
make build

# Flash to USB (replace /dev/sdX)
dd if=release/leenium-*.iso of=/dev/sdX bs=4M status=progress conv=fsync

Boot from USB and follow the installer. Drive gets wiped and partitioned automatically.

Method 2 — From a live Arch ISO

Wipes drive

Boot the official Arch ISO, clone leenium at the live shell, run the installer. It handles disk setup from there.

# From the Arch live shell:
git clone https://github.com/leenium/leenium
cd leenium
chmod +x install.sh
./install.sh

Drive gets wiped. Back up anything you care about first.

Method 3 — On top of an existing Arch install

Preserves partitions

Already have Arch set up? Run Leenium on top. Use archinstall for a minimal base if you don't have one yet — pick btrfs, limine, no desktop profile.

# archinstall settings:
#   Filesystem:  btrfs
#   Bootloader:  limine
#   Profile:     minimal (no desktop)
#   User:        non-root with sudo
archinstall

After rebooting into Arch:

git clone https://github.com/leenium/leenium
cd leenium
chmod +x install.sh
./install.sh

Partitions stay. Leenium just layers on top.

What you need (methods 2 + 3)

System

x86_64
btrfs root
limine bootloader
Secure Boot off

Environment

Normal user with sudo
Internet connection
No GNOME or KDE installed
Fresh Arch, not a derivative

After install

Log in → Super+K for keybindings
leenium-update to get latest
post-install guide for what's next
