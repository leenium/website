First Boot

What happens right after install.

Leenium runs first-login automation once, then expects you to confirm update state and environment basics.

Recovery commands Manual Home

Automatic first-run behavior

Triggered by marker:

~/.local/state/leenium/first-run.mode
  • Power profile selection and battery monitor setup
  • UFW defaults and Docker network integration
  • Resolver setup for /etc/resolv.conf
  • Desktop cleanup, icon cache refresh, service activation

Immediate actions

  • leenium-launch-wifi
  • leenium-update
  • Super + K for the keybinding cheat sheet
  • leenium-theme-list and leenium-font-list

Appearance controls

Use Super + Space then Appearance, or direct commands:

  • leenium-theme-set <name> and leenium-theme-list
  • leenium-font-list
  • Background selector reads ~/Pictures/Wallpapers/
  • leenium-plymouth-reset resets unlock screen

Optional setup commands

leenium-setup-fingerprint leenium-setup-fido2 leenium-install-dev-env leenium-install-terminal leenium-install-vscode leenium-install-docker-dbs

If first run did not complete

leenium-first-run
