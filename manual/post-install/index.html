First Boot

What happens right after install.

Leenium runs first-login automation once, then expects you to confirm update state and environment basics.

Automatic first-run behavior

Triggered by marker:

~/.local/state/leenium/first-run.mode
  • Power profile selection and battery monitor setup
  • UFW defaults and Docker network integration
  • Resolver setup for /etc/resolv.conf
  • Desktop cleanup, icon cache refresh, service activation

Immediate actions

  • leenium-launch-wifi
  • leenium-update
  • Super + K for the keybinding cheat sheet
  • leenium-theme-list and leenium-font-list

Appearance controls

Use Super + Space then Appearance, or direct commands:

  • leenium-theme-set <name> and leenium-theme-list
  • leenium-font-list
  • Background selector reads ~/Pictures/Wallpapers/
  • leenium-plymouth-reset resets unlock screen

Optional setup commands

leenium-setup-fingerprint leenium-setup-fido2 leenium-install-dev-env leenium-install-terminal leenium-install-vscode leenium-install-docker-dbs

Daily command groups

  • leenium-launch-* for Wi-Fi, Bluetooth, audio, browser, editor, and UI entry points
  • leenium-update* for updates and update log analysis
  • leenium-refresh-* to re-apply shipped defaults for selected components
  • leenium-restart-* to restart individual desktop services
  • leenium-theme-* and leenium-font-* for appearance control

Health and diagnostics

leenium-health
leenium-debug
leenium-debug --print
leenium-upload-log this-boot

Power, battery, and mobility helpers

  • leenium-battery-status, leenium-battery-remaining, and leenium-battery-monitor
  • leenium-brightness-display and leenium-brightness-keyboard
  • leenium-hibernation-available, leenium-hibernation-setup, and leenium-hibernation-remove
  • leenium-hyprland-monitor-* helpers for display switching and scaling cycles

If first run did not complete

leenium-first-run
