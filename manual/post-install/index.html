Leenium Post-Install Guide

What Leenium does automatically on first login, what to do next, and which optional setup commands matter once the system is already usable.

What Runs Automatically

On first login, Hyprland autostart runs leenium-cmd-first-run once, triggered by this marker:

~/.local/state/leenium/first-run.mode

System setup

  • Power profile selection
  • Battery monitor timer setup
  • UFW defaults and Docker integration
  • /etc/resolv.conf resolver setup

Desktop cleanup

  • GNOME interface defaults
  • Icon cache refresh
  • Elephant user service activation
  • Installer sudoers cleanup and critical notifications

Immediate Actions

  1. Connect networking if needed with leenium-launch-wifi
  2. Run a full update with leenium-update
  3. Open the keybindings cheatsheet with Super + K
  4. Check theme and font state with leenium-theme-list and leenium-font-list

If the machine feels "mostly installed but slightly off," run leenium-update before changing anything. Stale package state causes most false alarms.

Appearance

All appearance controls live under Super + Space → Appearance, or via the commands below.

Themes & visuals

  • leenium-theme-set <name> — switch theme
  • leenium-theme-list — list available themes
  • leenium-font-list — list available fonts

Backgrounds & unlock

  • Appearance → Background — pick from theme or ~/Pictures/Wallpapers/
  • Appearance → Unlock — set Plymouth unlock screen per theme
  • leenium-plymouth-reset — revert to default unlock screen

Drop your own wallpapers into ~/Pictures/Wallpapers/ — they appear automatically in the background selector.

Optional Setup

leenium-setup-fingerprint leenium-setup-fido2 leenium-install-dev-env leenium-install-terminal leenium-install-vscode leenium-install-docker-dbs

If First Boot Did Not Finish

Manually re-run the first-run sequence:

leenium-cmd-first-run

Then continue with the Troubleshooting chapter if the result still looks wrong.
