Common symptom map

  • Launcher not opening: run leenium-restart-walker, then verify Hypr bindings from keybindings docs
  • Bar missing after login: run leenium-restart-waybar and inspect journalctl --user output
  • Notifications broken: run leenium-restart-mako and verify notification daemon conflicts
  • Theme mismatch after update: run leenium-theme-set <theme> then component-level leenium-refresh-* commands
  • External monitor layout issues: use leenium-hyprland-monitor-* helpers before manual monitor edits

Recovery order

  1. Read the relevant log file
  2. Run the smallest targeted fix
  3. Collect debug info before broad reset commands
  4. Escalate to reinstall-style commands only when needed

When to reinstall vs. when to patch

  • Patch in place when issue is isolated to one component or one config target
  • Use leenium-reinstall-configs only when multiple defaults drifted and targeted refresh is too slow
  • Use leenium-reinstall only after logs confirm broad payload inconsistency
  • Always capture debug output before broad reinstall actions for reproducibility
