Leenium Troubleshooting

When something breaks: logs first, smallest targeted recovery command second, broad resets last. Follow that order every time.

Install Failures

Always start here for install-time issues:

/var/log/archinstall/install.log
/var/log/leenium-install.log

To share the failure for support:

leenium-upload-log install

First-Run Problems

If the system boots but first-login automation did not complete:

leenium-cmd-first-run

Then verify the first-run marker state:

cat ~/.local/state/leenium/first-run.mode

UI Component Recovery

Restart services

  • leenium-restart-waybar
  • leenium-restart-walker
  • leenium-restart-mako

Reapply config

  • leenium-reinstall-configs

leenium-reinstall-configs is a reset-style tool. Use it when config drift is the likely cause, not as the first response to every UI problem. It can overwrite local customizations.

Update Recovery

  • leenium-update — normal update path
  • leenium-update-analyze-logs — when update logs mention initramfs or related warnings
  • leenium-reinstall — when the baseline package state needs to be forced back into line

Collect Debug Information

leenium-debug
leenium-debug --print
leenium-upload-log this-boot
leenium-upload-log last-boot

Recovery Order

  1. Read the relevant log file first
  2. Run the smallest targeted recovery command
  3. Collect debug output before any broad resets
  4. Use reinstall-style commands only after targeted options are exhausted
