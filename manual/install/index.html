Leenium install, stage by stage.

Preflight requirements

  • Vanilla Arch Linux on x86_64 with btrfs and limine
  • Normal sudo user (not root) and Secure Boot disabled
  • No pre-installed GNOME/KDE desktop state
  • No derivative distro baseline (Manjaro, Garuda, etc.)
If docs and source ever diverge, source is authoritative and docs should be updated.

Quick stats

  • 5 install stages
  • 2 primary log files
  • 1 completion marker

Install stages

  1. preflight - guard checks, migration, log setup, marker prep
  2. packaging - base packages, fonts, Neovim, hardware package sets
  3. config - defaults, theme setup, app/shell configs, hardware fixes
  4. login - Plymouth, SDDM, Limine, Snapper, mkinitcpio integration
  5. post-install - hibernation setup, cleanup, completion state

Supported install paths

  • Leenium ISO for the direct path
  • Official Arch ISO + cloned leenium repo
  • Existing Arch install layered with Leenium payload
Keep expectations aligned: Leenium assumes Arch baseline choices and applies an opinionated desktop layer on top.

Before running installer

git clone https://github.com/leenium/leenium
cd leenium
chmod +x install.sh
./install.sh

What each stage changes

  • preflight: validates platform assumptions, prepares logs, and enforces install guards
  • packaging: installs base package set plus hardware-aware package additions
  • config: applies defaults from Leenium config payload and system-level tweaks
  • login: configures session and boot UX pieces like SDDM and Plymouth integration
  • post-install: final cleanup, hibernation setup flow, and completion marker handling

Logs and completion markers

When an install fails, check logs first before trying broad changes.

/var/log/leenium-install.log
/var/log/archinstall/install.log
/var/tmp/leenium-install-completed

Fast triage

  • Installer does not start -> verify preflight requirements
  • Packages fail -> inspect /var/log/archinstall/install.log
  • Config step fails -> inspect /var/log/leenium-install.log
  • Boot works but desktop is off -> continue in Post-Install guide
Go to Post-Install

Escalation checklist

  1. Capture both install logs and exact failing stage
  2. Re-run with network verified and no background package manager activity
  3. Collect runtime state using leenium-debug --print after first boot attempts
  4. Escalate with command output plus logs instead of re-running blind resets
