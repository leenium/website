Scope

Leenium is the post-archinstall system layer. The main entrypoint is install.sh, designed to run from ~/.local/share/leenium (or the cloned repo directory) on a base Arch system.

If this page and the source code ever disagree, the code is authoritative. Update the docs.

Preflight Requirements

The installer checks these conditions before touching anything:

System

  • Vanilla Arch Linux
  • x86_64 architecture
  • btrfs root filesystem
  • limine bootloader present

Environment

  • Normal user with sudo, not root
  • Secure Boot disabled
  • No pre-installed GNOME or KDE
  • No derivative distro state (Manjaro, Garuda, etc.)

Install Stages

  1. preflight — guard checks, logging setup, migration execution, first-run marker prep, mkinitcpio hook deferral
  2. packaging — base packages, fonts, Neovim, icons, webapps, TUIs, hardware-specific package sets
  3. config — copy defaults, apply theme setup, deploy app and shell config, run hardware fixes
  4. login — Plymouth, SDDM, Limine, Snapper, mkinitcpio integration
  5. post-install — hibernation setup, pacman cleanup, reboot permissions, final completion state

Logs and Completion Markers

The first response to any install failure should be log review, not trial-and-error changes.

/var/log/leenium-install.log
/var/log/archinstall/install.log
/var/tmp/leenium-install-completed

Every stage script runs through logging wrappers. If a stage fails, the install log will show which script failed with timestamps. Start there before changing anything.

Fast Triage

  • Install never starts — check preflight requirements first.
  • Packages fail — inspect /var/log/archinstall/install.log.
  • Leenium config fails — inspect /var/log/leenium-install.log.
  • Install finished but first login looks wrong — see Post-Install Guide.
