Hyprland

Know what to edit and what not to touch.

Leenium has a layered Hyprland config. Edit user override files, and treat defaults as project-owned baseline.

Load order

~/.config/hypr/hyprland.conf sources defaults from ~/.local/share/leenium/default/hypr first, then user overrides from ~/.config/hypr.

Defaults: autostart, bindings, envs, looknfeel, input, windows, current theme
Overrides: monitors.conf, input.conf, bindings.conf, looknfeel.conf, autostart.conf
Practical rule: customize files under ~/.config/hypr.

User override tree

  • monitors.conf
  • input.conf
  • bindings.conf
  • looknfeel.conf
  • autostart.conf
  • hypridle.conf, hyprlock.conf, hyprsunset.conf, xdph.conf

Default tree

  • looknfeel.conf, input.conf, windows.conf, envs.conf
  • autostart.conf and bindings/
  • apps.conf and apps/*.conf
  • plain-bindings.conf for direct overrides

Useful commands while tweaking

hyprctl reload
leenium-menu-keybindings --print
leenium-reinstall-configs
leenium-reinstall-configs is a reset action and can overwrite project-owned configuration changes.

Safe customization workflow

  1. Edit only files under ~/.config/hypr for user overrides
  2. Reload live with hyprctl reload
  3. Validate bindings via leenium-menu-keybindings --print
  4. Commit your override files if you manage dotfiles in git

High-value override files

  • monitors.conf for layout, scaling, and monitor priority
  • bindings.conf for user shortcuts
  • looknfeel.conf for gaps, borders, blur, and visual behavior
  • autostart.conf for session startup process control
