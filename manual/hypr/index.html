Hyprland

Hypr Config Reference

Leenium uses a layered Hyprland setup. This page explains which files are project-owned, which are yours to edit, and what happens when Leenium helper commands touch them.

Load Order

Your ~/.config/hypr/hyprland.conf first sources Leenium defaults from ~/.local/share/leenium/default/hypr, then loads user overrides from ~/.config/hypr.

Defaults:  autostart, media / clipboard / tiling / utilities bindings,
           envs, looknfeel, input, windows, current theme.

Overrides: monitors.conf, input.conf, bindings.conf,
           looknfeel.conf, autostart.conf.

Practical rule: edit files under ~/.config/hypr. Treat ~/.local/share/leenium/default/hypr as project-owned baseline — it can be overwritten by leenium-reinstall-configs.

User Override Tree

Main overrides

  • monitors.conf
  • input.conf
  • bindings.conf
  • looknfeel.conf
  • autostart.conf

Related runtime files

  • hypridle.conf
  • hyprlock.conf
  • hyprsunset.conf
  • xdph.conf

Leenium Default Tree

  • looknfeel.conf, input.conf, windows.conf, envs.conf — baseline behavior
  • autostart.conf — starts base services and first-run entrypoints
  • bindings/ — media, clipboard, tiling, and utility bindings
  • apps.conf and apps/*.conf — app-specific window rules
  • plain-bindings.conf — simpler direct binding overrides

Commands When Tweaking

hyprctl reload
leenium-menu-keybindings --print
leenium-reinstall-configs

leenium-reinstall-configs is not a light refresh. It resets project-owned config files and can overwrite local customizations in the defaults tree. Use it intentionally.
